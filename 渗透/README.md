# 渗透

这个目录建议定位为“授权测试与漏洞验证学习”，重点是方法论、信息收集、漏洞成因理解和实验环境复盘。

## 边界说明

这里只适合整理授权环境、靶场和教学场景下的内容，不建议放未授权目标利用材料。

## 建议拆分方向

- 信息收集与资产识别
- Web 常见风险成因
- 中间件暴露面理解
- 靶场复盘与授权测试笔记
- 报告撰写与修复建议

## 推荐先学什么

1. 从资产和暴露面出发理解风险
2. 学会把漏洞复现写成防守复盘
3. 把测试结果转成修复建议
4. 强化报告与沟通能力

## 适合沉淀的内容

- 靶场复盘
- 授权测试 checklist
- 漏洞成因分析
- 修复建议模板
