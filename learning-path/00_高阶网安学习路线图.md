# 高阶网安学习路线图

这份路线图不是按“工具清单”来排，而是按真实能力成长来排。

## 阶段 1：打牢基础面

目标：能把一个安全问题放回系统上下文中理解，而不是只盯一个漏洞编号。

建议先系统补齐：

- 网络基础：TCP/IP、DNS、HTTP、TLS、代理与反向代理
- 操作系统：Linux 进程、权限、文件系统、systemd、日志；Windows 账户、服务、计划任务、注册表、事件日志
- Web 基础：认证、会话、上传、反序列化、模板、文件路径、代理链
- 开发基础：至少熟悉 Python，能读一点 Java、Go、PHP 或 JavaScript
- 数据基础：SQL、NoSQL、缓存、消息队列的基本行为

## 阶段 2：建立漏洞到防守的映射能力

目标：看到一个漏洞时，能立刻想到版本、配置、暴露面、日志、补丁与缓解。

重点训练：

- 读官方公告和补丁说明
- 识别影响范围和前置条件
- 判断暴露面是否真实存在
- 设计排查项：版本、端口、配置、日志、账户、文件
- 写出修复建议和验证步骤

这个阶段建议多做“防御研究笔记”，而不是堆 PoC。

## 阶段 3：进入蓝队与检测思维

目标：不只是知道风险，而是能把风险转成可执行的检测与响应流程。

需要建立的能力：

- 日志来源盘点
- 攻击链拆解
- IOC 与 TTP 区分
- 告警去噪与优先级排序
- 时间线分析
- 误报与漏报复盘

## 阶段 4：做安全工程化

目标：把个人技巧变成团队可重复使用的资产。

建议开始沉淀：

- 资产清单 schema
- 统一排查脚本
- 检测规则模板
- 安全基线 checklist
- 应急响应流程文档
- 复盘模板

## 阶段 5：形成自己的方向

当基础足够后，可以根据兴趣选方向纵深发展：

- 蓝队 / SOC / 威胁狩猎
- 检测工程
- 应急响应与取证
- 应用安全 / 代码审计
- 云安全 / IAM / K8s
- 中间件与基础设施安全
- 安全自动化平台

## 建议的长期习惯

- 每周精读一个高质量漏洞公告或补丁说明
- 每月把一个安全事件写成复盘
- 每季度整理一批自己的检测规则或排查脚本
- 持续维护自己的资产、日志、工具和知识库结构

## 常见误区

- 只追求 CVE 数量，不关注成因抽象
- 只会复现，不会排查与修复
- 只会工具命令，不理解系统原理
- 只会单机分析，不会做横向影响判断
- 只会个人技巧，不会沉淀成团队资产
