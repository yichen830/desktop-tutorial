# 蓝队与防御能力地图

这份文档更偏“岗位能力图谱”，适合已经入门、希望往更高阶走的人。

## 资产安全

需要掌握：

- 资产发现与归并
- 服务指纹和版本盘点
- 暴露面分析
- 互联网与内网边界识别
- 历史遗留系统治理

输出物通常包括：

- 资产清单
- 风险分级
- 高风险服务治理列表
- 基线偏差清单

## 检测工程

需要掌握：

- 数据源接入：Web、系统、认证、云日志、EDR、WAF
- 规则设计：行为、序列、阈值、画像、关联
- 告警调优：误报压制、优先级、去重、聚合
- 检测验证：样本、回放、对抗、覆盖评估

输出物通常包括：

- 检测规则
- 检测逻辑说明
- 误报处理策略
- 命中后的处置手册

## 威胁狩猎

需要掌握：

- 假设驱动分析
- 异常账号、异常进程、异常网络、异常任务的筛选
- 时间线串联
- 多源日志交叉验证
- 从弱信号中找强结论

## 应急响应

需要掌握：

- 现场保护
- 影响范围评估
- 账号与凭据处置
- 横向影响排查
- 持久化与后门清理
- 恢复与复盘

## 应用安全

需要掌握：

- 常见 Web 风险成因
- 代码审计基础
- 中间件配置与部署风险
- 安全测试与研发流程结合
- 修复建议能落到代码和配置层

## 安全工程化

需要掌握：

- 脚本化能力
- 数据清洗与格式统一
- 平台接入与 API 自动化
- 文档化与标准化
- 把一次经验沉淀成流程和模板

## 高阶能力的标志

如果你已经开始做这些事，通常说明你在往更高阶走：

- 会设计排查流程，而不是只跑工具
- 会读补丁和源码 diff，而不是只读二手文章
- 会和运维、研发、IT 协同推进修复
- 会把检测、修复、复盘串起来
- 会把临时经验沉淀成长期资产
