# 应急响应清单

这份清单偏中间件、Web、主机侧通用场景。

## 一、先做确认

- 事件来源是什么：告警、外部通报、人工发现、日志异常
- 涉及哪些资产
- 是否仍在持续发生
- 是否涉及高价值系统、账号或数据

## 二、先保护现场

- 不要急着大范围删文件
- 不要先重装再分析
- 先保留日志、进程、网络连接、任务、关键目录快照
- 记录当前时间、主机信息、排查人员和操作痕迹

## 三、快速判断影响面

- 同类系统是否共享版本和配置
- 是否存在横向访问痕迹
- 是否涉及统一认证、数据库、制品库、代码平台
- 是否存在批量异常登录、异常任务、异常进程

## 四、常见排查项

### Web / 中间件

- 访问日志
- 错误日志
- 反向代理日志
- 上传目录
- 应用配置文件
- 定时任务与脚本

### Linux 主机

- 新增用户与 sudoers 变化
- crontab / systemd / rc.local
- 可疑进程与父子进程链
- 可疑二进制与脚本
- 异常网络连接

### Windows 主机

- 新增本地用户与管理员组变化
- 计划任务、服务、启动项
- PowerShell 历史与脚本块日志
- 可疑父子进程链
- RDP 与登录事件

## 五、处置动作

- 临时隔离资产或入口
- 停用高风险账号
- 轮换相关凭据
- 清理持久化点
- 打补丁或临时缓解
- 对同类资产同步核查

## 六、恢复前必须确认

- 根因大致明确
- 后门与持久化已排查
- 高风险凭据已轮换
- 同类系统已同步处理
- 监控和告警规则已补上

## 七、复盘要回答的问题

- 首个异常是什么时候出现的
- 为什么没有更早发现
- 哪个控制点本可以拦住它
- 哪些资产或流程需要改
- 哪些脚本、规则、文档可以沉淀下来
